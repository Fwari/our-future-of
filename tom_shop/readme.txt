100421 톰샵 상점 프로그램 - (c)t0mcat

1. 개요
커뮤니티에서 사용할 수 있는 상점 프로그램 입니다. 상품의 등록,
수정, 삭제, 회원 등록, 상품 구매, 사용, 사용이력 확인 등의 기본 기능에
충실하였으며 관리자가 디자인을 수정하기 편리하도록 만들었습니다.
기본적으로 상점NPC가 있다는 개념이 적용되어 있습니다. 포인트(돈)을
기반으로 회원들이 상품을 구매하고 사용하는 이력을 자동으로 남김으로서
영자/총괄님들이 일일히 승인 후 처리를 하는 수고를 조금이나마 덜어줍니다.


2. 설치
(1) tom_shop.zip 파일의 압축을 풀면 안에 tom_shop 폴더가 있습니다.
(2) FTP 프로그램을 이용하여 본인께서 사용중이신 웹 계정에
접속하신 뒤 tom_shop 폴더를 통째로 업로드 합니다.
(3) 웹서버 내의 tom_shop 폴더 안의 data 폴더의 퍼미션(권한)을 777로 바꿉니다.
(4) data 폴더 안으로 들어가신 뒤,  폴더 안의 모든 txt 파일들의 퍼미션(권한)을
666으로 바꿉니다.
(5) lib.php의 7번째 줄에 관리자 패스워드가 있습니다. 이것을 수정해서 사용하세요.
(6) lib.php의 11번째 줄부터 28번째 줄 까지는 상점주인의 설명 말투입니다.
예시문으로 상점NPC가 설명하는 말투를 사용하였습니다만, 보통의 설명투로 쓰셔도 됩니다.

화면 왼쪽 하단의 t0mcat 카피라이트 왼쪽에 admin 글자를 누르면 관리자페이지로
들어갑니다. 관리자패스워드로 로그인 하신 뒤 회원들과 회원 각자의 패스워드, 기본포인트
등을 등록하여 최초 가입을 시켜 주세요.

3. 스킨 변경
사용자에게 보여지는 화면 파일 목록의 스킨을 고치시려면 이하의 파일을 수정 해 주세요.
menu.php, log_view.php, mine.php, shop.php, talk.php
talk.php - 왼쪽 상단의 대화창 부분.
menu.php - 왼쪽 하단의 메뉴 부분.
mine.php - 왼쪽 상단의 소지품창 부분
shop.php - 오른쪽의 상품목록 부분
이상 파일의 html 파트를 수정하실 수 있습니다. admin.php는 오른쪽의 관리자 부분이지만
수정을 권장하지 않습니다. 클래스 적용 되어 있는 CSS 파트를 수정하시려면 style.css를
수정 해 주세요. 


4. 관리자 사용 방법

 제일 먼저 화면 왼쪽 하단의 t0mcat 카피라이트 왼쪽에 admin 글자를 누르면 관리자페이지로
들어갑니다. 관리자패스워드로 로그인 하신 뒤 "신규 캐릭터 등록"으로 회원들과 회원 각자의
패스워드, 기본포인트 등을 등록하여 최초 가입을 시켜 주세요. 최초 가입을 위한 회원 각자의
패스워드는 쪽지로 수집하시길 권해드립니다. 회원용 비밀번호는 맨 윗줄의 글자수나 숫자,영문은
관계없지만 한글사용이나 띄어쓰기는 좋지 않습니다. 보안이 매우 낮으므로 타사이트에서 사용중인
패스워드 보다는 단순하지만 남이 쉽게 알지는 못하는 것을 권장합니다.

"캐릭터 정보수정" 으로 이미 등록한 캐릭터의 정보와 소지중인 아이템을 수정할 수 있습니다.

"상점 물품 수정" 에서 상점에서 판매중인 아이템의 이름, 이미지, 가격, 설명 등을 수정하실 수
있습니다. 이미지는 img 폴더에 FTP로 직접 수동으로 업로드 해 주신 뒤 파일명을 기입 해 주세요.
작성방법에 대한 가이드는 해당 메뉴 하단에 설명 해 두었습니다.

5. 사용자 사용 방법

 상점 메인에서 오른쪽에 보이는 상품명을 클릭하면 상품에 대한 설명과 가격을 왼쪽 상단에서
설명합니다. 그 상태에서 자신의 캐릭터의 이름을 선택하시고 본인의 고유 패스워드를 입력하시면
물건을 구매하실 수 있습니다.

"소지품 확인/사용" 에서 본인의 캐릭터의 이름을 선택하시고 패스워드를 입력하시면 이미 구매한
상품과 현재 보유중인 포인트를 확인하실 수 있습니다. 보유중인 상품이 있다면 사용하실 수 있습니다.

"상점 기록 보기"에서는 회원 전체가 그동안 어떤 아이템을 구매하고 사용하였는지 이력이 나옵니다.
관리자가 수정했을시에도 관리자수정, 관리자 등록 이라고 이력이 남아서 모든 회원의 이력 상황을
투명하게 열람 및 관리 할 수 있습니다.
